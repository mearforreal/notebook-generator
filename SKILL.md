# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm run dev       # Start development server (http://localhost:3000)
npm run build     # Production build
npm run start     # Start production server
npm run lint      # ESLint
```

No test suite exists.

## Environment

Requires `ANTHROPIC_API_KEY` in `.env.local`. The app will fail silently on tutorial generation without it.

## Architecture

Next.js 16.1.6 App Router application (React 19, TypeScript, Tailwind v4). The primary interface is `/administrator` — a five-tab dashboard that orchestrates the full pipeline: **CSV → Notebook → Tutorial → Charts**.

### Core Pipeline

1. **CSV Upload** (`app/api/upload-csv/route.ts`) — validates and stores CSVs to `app/ipynb/uploads/` with `{timestamp}_{sanitized_name}.csv` naming.
2. **Notebook Builder** (`app/api/generate-notebook/route.js` + `lib/custom-notebook-generator.js`) — generates a 10-cell Jupyter notebook (.ipynb) into `app/ipynb/{project-slug}/` from a CSV. The notebook covers: imports, load, overview, missing values, duplicates, type conversion, outlier detection, export.
3. **Tutorial Builder** (`app/api/generate-tutorial/route.js`) — reads an existing .ipynb, calls the Claude API (claude-haiku-4-5) to generate 2–4 sentence explanations per code cell, then writes a full React page.js to `app/ipynb/{project-slug}/tutorial/page.js`. The generated page is immediately routable at `/ipynb/{project-slug}/tutorial`.
4. **Charts Builder** (`app/api/generate-charts/route.js`) — generates Chart.js visualizations from cleaned CSVs.
5. **Notebook Runner** (`app/api/run-notebook/route.js`, `app/api/jupyter-execute/route.js`) — executes notebooks via a local Jupyter server managed by `lib/jupyter-server.js`.

### Directory Layout Under `app/ipynb/`

```
uploads/     ← raw uploaded CSVs
notebooks/   ← .ipynb files (legacy path from upload route)
cleaned/     ← cleaned CSVs output by notebook execution
{project}/   ← per-project folder created by generate-notebook
  *.ipynb
  tutorial/page.js   ← generated by tutorial builder
```

### Key Patterns

**Claude API calls** live only in `app/api/generate-tutorial/route.js`. The `generateExplanation()` function wraps `anthropic.messages.create()` with claude-haiku-4-5. All other routes are pure file I/O or notebook manipulation.

**Generated React pages** are written to the filesystem as `.js` files and served by Next.js App Router's filesystem routing — no build step needed after generation; Next.js picks them up on the next request in dev mode. In production, a rebuild is required.

**Path construction** always uses `path.join(process.cwd(), 'app', 'ipynb', ...)` — never relative paths.

**Mixed TypeScript/JavaScript** — the upload route and `lib/` utilities are `.ts`; all `components/administrator/` and `app/api/` files (except upload) are `.js`/`.jsx`. Don't convert JS files to TS without a deliberate decision to migrate the whole area.

**Tailwind v4** is used (PostCSS plugin via `@tailwindcss/postcss`). The config is in `tailwind.config.ts`.
